# Search-First Workflow

コードを書く前に、以下の順序で調査する。

## 1. 既存コード検索
grep / find で関連する既存実装を探す。

## 2. パターン確認
既存コードベースのパターンに合わせる。

## 3. 依存関係確認
package.json の既存依存で代替できないか確認。

## 4. ドキュメント・ルール確認
CLAUDE.md、.claude/rules/、.rwl/lessons/ を確認。

## 5. 実装
上記1-4の調査結果を踏まえて初めてコードを書く。

## アンチパターン
- 調査せずにいきなりコードを書く
- 既存パターンと異なるスタイルで書く
- 既存の依存関係にあるライブラリを重複インストールする
- 既に同じ機能の実装があるのに新たに作る
- 同名・類似機能の実装があるのに抽象ユーティリティを新設する
